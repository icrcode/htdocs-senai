Perguntas Frequentes — Windows

Dúvidas comuns sobre instalação, configuração e uso do XAMPP.

Instalação e Configuração

📦 Como instalo o XAMPP? ▼

O XAMPP para Windows existe em três formatos:

  • Installer — a forma mais fácil. Execute o instalador e siga o assistente.
  • ZIP — extraia na pasta desejada e execute setup_xampp.bat.
  • 7zip — versão compactada para conexões lentas.

Após a instalação, use o XAMPP Control Panel para iniciar/parar Apache, MySQL, FileZilla e Mercury.

🗄️ O XAMPP inclui MySQL ou MariaDB? ▼

Desde as versões 5.5.30 e 5.6.14, o XAMPP usa MariaDB em vez de MySQL. Os comandos e ferramentas são idênticos para ambos.

🚀 Como iniciar e parar o XAMPP? ▼

Use o XAMPP Control Panel: \xampp\xampp-control.exe

Ou via arquivos batch:

  • Iniciar tudo: \xampp\xampp_start.exe
  • Parar tudo: \xampp\xampp_stop.exe
  • Apache iniciar/parar: \xampp\apache_start.bat / apache_stop.bat
  • MySQL iniciar/parar: \xampp\mysql_start.bat / mysql_stop.bat
✅ Como verificar se tudo está funcionando? ▼

Digite no navegador: http://localhost/ ou http://127.0.0.1/

Você verá a página inicial do XAMPP se tudo estiver correto.

⚙️ Como instalar um servidor como serviço do Windows? ▼

Via XAMPP Control Panel (requer Administrador), ou pelos scripts:

  • Instalar Apache: \xampp\apache\apache_installservice.bat
  • Remover Apache: \xampp\apache\apache_uninstallservice.bat
  • Instalar MySQL: \xampp\mysql\mysql_installservice.bat
  • Remover MySQL: \xampp\mysql\mysql_uninstallservice.bat

Segurança

🔒 O XAMPP é pronto para produção? ▼

Não. O XAMPP é exclusivamente para desenvolvimento local. Vulnerabilidades padrão:

  1. O usuário MySQL root não tem senha.
  2. O daemon MySQL é acessível via rede.
  3. O ProFTPD usa senha pública "lampp".
  4. Usuários padrão de Mercury e FileZilla são públicos.

Para proteger, execute o console de segurança em http://localhost/security/.

🔑 Como definir uma senha root no MySQL? ▼

Abra o XAMPP Shell no Control Panel e execute:

mysqladmin.exe -u root password suasenha

Atualize também \xampp\phpMyAdmin\config.inc.php com a nova senha.

🌐 Como habilitar acesso remoto ao phpMyAdmin? ▼

Atenção: habilitar acesso externo é um risco de segurança. Use só em desenvolvimento local controlado.

Edite apache\conf\extra\httpd-xampp.conf e substitua Require local por Require all granted na diretiva do phpMyAdmin. Reinicie o Apache.

Desenvolvimento e PHP

📁 Onde colocar os arquivos do meu projeto? ▼

O diretório raiz é \xampp\htdocs. Exemplos de acesso:

  • htdocs\test.html → http://localhost/test.html
  • htdocs\test.php → http://localhost/test.php
  • htdocs\meu-projeto\ → http://localhost/meu-projeto/
🐘 Por que mudanças no php.ini não surtiram efeito? ▼

O PHP pode estar usando um arquivo diferente. Verifique em PHPInfo qual arquivo está carregado (campo "Loaded Configuration File").

Lembre: sempre reinicie o Apache após editar o php.ini.

✉️ Como configurar envio de e-mail com XAMPP? ▼

No php.ini, seção [mail function], adicione:

sendmail_path = "\"C:\xampp\sendmail\sendmail.exe\" -t"

No sendmail.ini, seção [sendmail]:

smtp_server=smtp.gmail.com
smtp_port=465
smtp_ssl=auto
auth_username=seu@gmail.com
auth_password=sua-senha

Reinicie o Apache. Use mail() normalmente no PHP.

🔌 Como ativar extensões PHP (ex.: IMAP)? ▼

Abra \xampp\php\php.ini, localize a linha com a extensão desejada e remova o ; do início:

extension=php_imap.dll

Reinicie o Apache. O mesmo processo vale para qualquer extensão desativada.

Resolução de Problemas

⚠️ Por que o Apache não inicia? ▼

Causas mais comuns:

  • Outro serviço (IIS, Skype) ocupa a porta 80 — erro: make_sock: could not bind to address 0.0.0.0:80
  • Firewall bloqueando a porta.
  • Antivírus interferindo nas conexões.

Dica: use xampp-portcheck.exe para verificar conflitos de porta.

🪟 Como usar o XAMPP na porta 80 no Windows 10/11? ▼

O Windows pode iniciar o IIS na porta 80. Para desativar:

  1. Abra Gerenciamento do Computador → Serviços.
  2. Localize World Wide Web Publishing Service.
  3. Pare o serviço e mude o tipo de inicialização para Desabilitado.
🦠 Meu antivírus detectou vírus no XAMPP — é real? ▼

Provavelmente falso positivo. Arquivos como xampp-manager.exe são marcados erroneamente por alguns antivírus. Todas as versões do XAMPP são verificadas com o Kaspersky Online Virus Scanner antes do lançamento. Verifique também com o Virus Total.

📋 Onde ficam os arquivos de configuração? ▼
  • Apache: \xampp\apache\conf\httpd.conf
  • PHP: \xampp\php\php.ini
  • MySQL: \xampp\mysql\bin\my.ini
  • FileZilla: \xampp\FileZillaFTP\FileZilla Server.xml
  • Sendmail: \xampp\sendmail\sendmail.ini
  • Mercury: \xampp\MercuryMail\MERCURY.INI
↔️ Posso mover a instalação do XAMPP? ▼

Sim. Após mover o diretório, execute setup_xampp.bat para atualizar os caminhos. Se tiver serviços Windows instalados, desinstale-os antes de mover e reinstale depois.
